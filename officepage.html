정밀 영역 탐지 및 데이터 식별

지능형 영역 탐지 및 정밀 크롭

시각적 노이즈 제거를 통한 분석 효율 극대화

방대한 도면 데이터 중에서 실제 분석이 필요한 BOM 영역만을 AI가 스스로 찾아내어 정밀하게 크롭(Crop)합니다.

  • 해상도 독립적 처리 : 도면의 크기나 화질에 관계없이 안정적으로 표 영역을 타기팅합니다.
  • 분석 부하 최적화 : 전체 도면이 아닌 핵심 영역만 분석함으로써 서버 리소스를 절약하고 처리 속도를 비약적으로 높입니다.
  • 정확도 향상 : 주변 설계 기호나 주석 등의 간섭을 원천 차단하여 오인식률을 최소화합니다.

NED 기반 데이터 식별 및 정합성

지능형 엔티티 인식을 통한 데이터 신뢰도 확보

단순히 글자를 읽는 수준을 넘어, “NED (Named Entity Detection)” 기술을 통해 추출된 데이터의 속성을 명확히 정의합니다.

SJN6004-064와 같은 유사한 형식의 코드라도 문맥을 분석하여 [품번]인지 [도면번호]인지 구분해냅니다.
  • 정합성 보장 : 비즈니스 로직에 기반한 검증으로 데이터 무결성을 유지합니다.
  • 프로세스 투명성 : 분석 과정의 Base64 처리 로그와 디버깅 정보를 제공하여 시스템 신뢰도를 높입니다.
  • 오류 자가 진단 : 유효하지 않은 항목 추출 시 즉각적인 디버그 피드백을 생성합니다.

도면 데이터 지능형 추출 프로세스

01
도면 업로드
이미지(JPG/PNG) 또는 PDF 형태의 원본 도면을 시스템에 입력
02
영역 자동 탐지
AI 에이전트가 도면 내 BOM 표의 위치를 1초 내에 정밀 식별
03
지능형 추출
VLM이 표 안의 텍스트와 구조를 해석하여 정형 데이터로 변환
04
검증 및 연동
추출 데이터 정제 후 ERP/PLM 시스템으로 자동 전송
이 모든 과정은 단 수초 만에 수행되며, 사용자 확인을 거친 후 즉시 자산화 시킬 수 있습니다.

OfficeGPT CAD BOM Agent는 한국어, 영어, 특수 기호 및 숫자가 복합하게 결합된 비정형 문자열 환경에서도 품번, 품명, 수량, 부품번호 등 핵심 엔티티를 정밀하게 식별합니다. 특히 다국어 혼용 및 제조사별 고유 코드 체계를 완벽하게 이해하여, 수작업 시 발생하기 쉬운 오타와 누락을 원천 차단하고 기업의 도면 데이터 자산화를 가속합니다.

시스템 확장성 및 보안 강화

Enterprise 연동

추출된 BOM 데이터는 사내 ERP, PLM, MES 시스템과 API를 통해 실시간 연동됩니다. 더 이상 엑셀 파일 업로드 과정조차 필요하지 않은 완전 자동화된 데이터 파이프라인을 구축할 수 있습니다.

로컬 VLLM 기반의 완전 보안

OfficeGPT는 외부 API를 전혀 활용하지 않는 로컬 VLLM(On-Premise) 엔진으로 동작합니다. 모든 도면 분석과 데이터 처리가 기업 내부 네트워크 내에서 완료되는 구조이므로, 민감한 설계 자산이 외부로 노출될 위험을 원천 차단합니다.
